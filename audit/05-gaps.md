# Track 5 — Gap Analysis

READ-ONLY audit of the ABC Kids HR & Payroll app. Scope: admin app (`app/index.html`, ~13.7k lines), contractor portal (`portal/index.html`, ~2.3k lines), 10 Deno edge functions (`supabase/functions/*`), and schema (`schema/schema.sql` + `schema/migrations/*`). This track covers **absence** only — what is NOT there. Track 2 covers existing-but-bad screens; Track 3 covers flow/algorithm correctness; Track 4 covers schema integrity.

**Legend:** OBSERVED = confirmed by reading code/grep. INFERRED = deduced from absence of references + structure. ASSUMPTION = an open judgment call about intended scope. "Evidence of absence" lists the grep terms / files searched that returned nothing (or only adjacent hits).

Goals: G1 pay PHP via Wise from Hubstaff time · G2 Hubstaff sync (CSV + cron) · G3 biweekly payroll lifecycle · G4 client invoicing (per-hour + per-session) · G5 onboarding (sign→profile→docs→review/countersign) · G6 contractor portal · G7 multi-company admin scoping · G8 document/compliance expiry · G9 security (RLS, caller-auth, payment lock).

## Prioritized list

### BLOCKERS (a goal cannot be met as shipped)

1. **Email notification pipelines dead in prod (no secrets)** — G5/G8. Expiry + hiring-doc-review digests and new-hire emails never send. Compliance/onboarding tracking is passive. *Fix is operational (set Resend + Gmail secrets), not code — but until then G8's reminder loop does not run.* (Evidence: `documents-expiry-check/index.ts:144`, `hiring-docs-review-check/index.ts:190`, WORKFLOWS §9.)
2. **No invoice delivery to client** — G4. Invoices generate but "sent" is a status lie; `companies.contacts` captured but unused. (Evidence: no `sendInvoice` in `app/index.html`; WORKFLOWS §6 "read-only AR".)
3. **No service-session bulk import** — G4. Per-session billing is unusable at real visit volume; the import columns exist but no importer. (Evidence: only single-row `service_sessions.insert` at 12939.)
4. **No invoice settlement data model (amount_paid/paid_at/due_date)** — G4. AR aging/overdue/partial-payment impossible. (Evidence: `2026-06-10_invoicing.sql` column list.)
5. **No contractor payslip PDF/print** — G6. Contractors have no portable pay record. (Evidence: zero `jsPDF`/`print` in `portal/index.html`.)

### NICE-TO-HAVES (ordered by impact)

6. No accounts-receivable / client-revenue rollup (G4).
7. No bulk approve (time entries) and no bulk document review (G3/G5/G8) — friction scales with headcount.
8. No per-contractor admin payslip statement (period print exists) (G3).
9. No confirmation prompts on invoice generate/void/mark-paid — policy gap vs CLAUDE.md money-action rule (G4).
10. No 13th-month / annual summary report (accrual exists) (G3).
11. `portal_notifications` + `shift_schedule` orphaned dead schema; no portal notifications surface (G6).
12. No structured offboarding / final-pay / termination-reason model (basic deactivate works) (G3/G6).
13. Portal lacks skeleton loaders, toast confirmations, optimistic rollback (G6 UX).
14. `invoice_no` not uniquely constrained + non-atomic allocation (G4 correctness — overlaps Track 4).
15. No multi-currency invoicing, no partial-payment, no in-portal expiry alert to contractor (G4/G8).
16. No client-facing identity/portal model (likely out of scope) (G4).

## Assumptions & open questions

- **ASSUMPTION:** Email digests are "blocked, not broken" — code is correct, prod secrets are unset (per WORKFLOWS §9 + MEMORY `project_email_notifications`). If secrets were set in prod after the docs were written, gap #1 downgrades to nice-to-have. **Open question:** are `RESEND_API_KEY` / Gmail app-password set in Supabase secrets now?
- **ASSUMPTION:** Clients are billed out-of-band (no client login expected), so "no client portal" and "no client identity model" are nice-to-have, not blockers. **Open question:** is electronic invoice delivery (email/PDF link) a required G4 deliverable, or is print-and-send manual acceptable to the owner?
- **ASSUMPTION:** per-session billing is intended for clients with potentially many visits per period; if real-world session counts are small, the "no bulk import" gap softens from BLOCKS to nice-to-have. **Open question:** expected sessions/client/period?
- **OBSERVED (positive, scope note):** The prior CRITICAL caller-auth hole in `wise-payouts` (MEMORY `project_security_audit_2026_06_06`) is now CLOSED — `wise-payouts/index.ts:253-296` enforces authenticated-admin (or cron-secret) on every action, with owner-only on staging transfers. G9 caller-auth is no longer an absence here. (Other G9 items — CSP, portal login rate-limit, `signature_data` XSS — are existing-but-weak, i.e. Track 2/3 territory, not absence.)
- **Coordination:** Items flagged "overlaps Track 3/4" (`invoice_no` uniqueness; PHS pay-calc modeling) are correctness/schema-integrity questions that those tracks own; listed here only where the *absence* of a column/constraint is the root cause.
- **NOT independently verified at byte level:** the exact set of confirmation prompts inside the Invoicing component (B row 4) and the portal's display of `review_reason` for rejected docs (B row 7) — flagged INFERRED/verify; Track 2's screen-by-screen pass should confirm.
